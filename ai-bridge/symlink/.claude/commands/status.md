Render a **status board** across this bundle's projects and (re)write it to
`DASHBOARD.md` at the bundle root. This is a **monitor, not a driver** — it reads
task state and reports; it **never** refines, promotes `draft → ready`, dispatches
agents, or merges. To advance work use `/pm-loop`; for ad-hoc parallel asks use
`/fanout`. Safe to run anytime, **including while a `/pm-loop` is active** — it only
reads task docs (+ light `gh` reconciliation) and rewrites the derived board.

> **Generic template file** (symlinked from the `ai-bridge` template). Reads the
> bundle's own `SCHEMA.md` for the task lifecycle and `instance.config.json` for
> values — never hardcode org/repo/path literals here.

## Input
`$ARGUMENTS`:
- **empty** → the full board (all projects).
- **`mine`** → only the "🔴 Awaiting you" section — fast triage of your queue.
- **`<project-slug>`** → scope to a single project.

## Steps

1. **Enumerate tasks.** Glob `projects/*/tasks/*.md`; read each one's frontmatter
   (`status`, `kind`, `assignee`, `open_questions`, `pr`, `depends_on`, `objective`,
   `target_repo`). Group by project. (This is the same state the PM reads in its
   Orient step — you are re-deriving the board, not changing anything.)

2. **Reconcile in-flight PRs (light, best-effort).** For tasks carrying a `pr:`,
   check real state with `gh pr view <url> --json state,mergeStateStatus` so "in
   flight" reflects reality, not a stale doc. If `gh` is unavailable/offline, fall
   back to the doc status and **note that the board may be stale** — never block on it.

3. **Bucket every task** by what it needs (see the `SCHEMA.md` lifecycle):
   - **🔴 Awaiting you** — a human decision is the only thing unblocking it:
     - `draft`, refined & clean (empty `open_questions`) → **approve** (`draft → ready`);
     - `draft` with non-empty `open_questions` → **answer** (list them by number);
     - `in-review` → **merge** the PR(s);
     - `blocked` → **unblock**.
     Tag each item with its verb.
   - **🟡 In flight** — `in-progress` (show `@assignee` + `target_repo`) and
     `in-review` (show PR link). *(`in-review` also appears under Awaiting you as a
     merge — intentional: it's both in flight and your decision.)*
   - **🟢 Next** — `ready` (dispatchable on the next tick) and `draft` still
     unrefined (empty `acceptance_criteria` — awaiting the PM's refine).
   - **⛔ Blocked** — `blocked`, with the reason and any unmet `depends_on`.
   - Leave `done` / `cancelled` off the board — tail-count them only.

4. **Write `DASHBOARD.md`** at the bundle root using the Layout below: absolute
   `/projects/...` links, most-urgent section first, `date -u +%Y-%m-%dT%H:%M:%SZ`
   as the refresh stamp. `DASHBOARD.md` is **derived and gitignored** — regenerate
   it freely and **never hand-edit** it.

5. **Print** the board (or the requested slice) to the user, leading with
   "🔴 Awaiting you" so the action queue is first. **Do not commit** — the board is
   gitignored, so there's nothing to stage; it's a view, not tracked state.

## Layout (write this to DASHBOARD.md)

```markdown
# Control Center — Dashboard

Status board across all projects, grouped by what needs attention. **Derived and
gitignored — do not hand-edit.** Regenerated by `/status` and at the end of each
`/pm-loop` tick from `projects/*/tasks/*.md`. Last refreshed: <ISO 8601>.

## 🔴 Awaiting you (<n>)
_A human decision is the only thing unblocking these._
* **approve** — [<task title>](/projects/<slug>/tasks/<id>.md) · draft refined, promote `draft → ready`
* **answer** — [<task title>](/projects/<slug>/tasks/<id>.md) · Q1: <question>; Q2: <question>
* **merge** — [<task title>](/projects/<slug>/tasks/<id>.md) · [<repo>#<n>](<pr-url>)
* **unblock** — [<task title>](/projects/<slug>/tasks/<id>.md) · <blocker reason>

## 🟡 In flight (<n>)
* [<task title>](/projects/<slug>/tasks/<id>.md) · in-progress · @<assignee> · `<org>/<repo>`
* [<task title>](/projects/<slug>/tasks/<id>.md) · in-review · [<repo>#<n>](<pr-url>)

## 🟢 Next (<n>)
_Dispatchable on the next tick, or awaiting the PM's refine._
* [<task title>](/projects/<slug>/tasks/<id>.md) · ready · @<assignee>
* [<task title>](/projects/<slug>/tasks/<id>.md) · draft (unrefined)

## ⛔ Blocked (<n>)
* [<task title>](/projects/<slug>/tasks/<id>.md) · <why> · depends_on <unmet paths>

---
Projects: <active count> · Open tasks: <count> · Done: <count> · Cancelled: <count>
```

Keep a section even when empty — render it with `_None._` so the board's shape is
stable (and the SessionStart hook reliably finds "Awaiting you").

## Notes
- **Link PRs, never bare URLs.** Render every PR as `[<repo>#<n>](<pr-url>)` — the
  **bare repo name** (not `<org>/<repo>`) as the link text, the PR URL as the target
  — per the instance `CLAUDE.md` "Reporting progress" rule. Same for any other
  artifact you cite.
- **Report-only.** Never promote `draft → ready`, never merge, never dispatch agents
  — those are `/pm-loop`'s job and the human's gates.
- **One instance.** A cross-instance ("all my groups") view is out of scope — run
  `/status` inside each instance. `DASHBOARD.md` is per-instance.
- No customer PII in the board.
