Render a **status board** across this bundle's projects and (re)write it to
`DASHBOARD.md` at the bundle root. This is a **monitor, not a driver** вАФ it reads
task state and reports; it **never** refines, promotes `draft вЖТ ready`, dispatches
agents, or merges. To advance work use `/pm-loop`; for ad-hoc parallel asks use
`/fanout`. Safe to run anytime, **including while a `/pm-loop` is active** вАФ it only
reads task docs (+ light `gh` reconciliation) and rewrites the derived board.

> **Generic template file** (symlinked from the `ai-bridge` template). Reads the
> bundle's own `SCHEMA.md` for the task lifecycle and `instance.config.json` for
> values вАФ never hardcode org/repo/path literals here.

## Input
`$ARGUMENTS`:
- **empty** вЖТ the full board (all projects).
- **`mine`** вЖТ only the "рЯФі Awaiting you" section вАФ fast triage of your queue.
- **`<project-slug>`** вЖТ scope to a single project.

## Steps

1. **Enumerate tasks.** Glob `projects/*/tasks/*.md`; read each one's frontmatter
   (`status`, `kind`, `assignee`, `open_questions`, `pr`, `depends_on`, `objective`,
   `target_repo`). Group by project. (This is the same state the PM reads in its
   Orient step вАФ you are re-deriving the board, not changing anything.)

2. **Reconcile in-flight PRs (light, best-effort).** For tasks carrying a `pr:`,
   check real state with `gh pr view <url> --json state,mergeStateStatus` so "in
   flight" reflects reality, not a stale doc. If `gh` is unavailable/offline, fall
   back to the doc status and **note that the board may be stale** вАФ never block on it.

3. **Bucket every task** by what it needs (see the `SCHEMA.md` lifecycle):
   - **рЯФі Awaiting you** вАФ a human decision is the only thing unblocking it:
     - `draft`, refined & clean (empty `open_questions`) вЖТ **approve** (`draft вЖТ ready`);
     - `draft` with non-empty `open_questions` вЖТ **answer** (list them by number);
     - `in-review` вЖТ **merge** the PR(s);
     - `blocked` вЖТ **unblock**.
     Tag each item with its verb.
   - **рЯЯ° In flight** вАФ `in-progress` (show `@assignee` + `target_repo`) and
     `in-review` (show PR link). *(`in-review` also appears under Awaiting you as a
     merge вАФ intentional: it's both in flight and your decision.)*
   - **рЯЯҐ Next** вАФ `ready` (dispatchable on the next tick) and `draft` still
     unrefined (empty `acceptance_criteria` вАФ awaiting the PM's refine).
   - **вЫФ Blocked** вАФ `blocked`, with the reason and any unmet `depends_on`.
   - Leave `done` / `cancelled` off the board вАФ tail-count them only.

4. **Write `DASHBOARD.md`** at the bundle root using the Layout below: absolute
   `/projects/...` links, most-urgent section first, `date -u +%Y-%m-%dT%H:%M:%SZ`
   as the refresh stamp. `DASHBOARD.md` is **derived and gitignored** вАФ regenerate
   it freely and **never hand-edit** it.

5. **Print** the board (or the requested slice) to the user, leading with
   "рЯФі Awaiting you" so the action queue is first. **Do not commit** вАФ the board is
   gitignored, so there's nothing to stage; it's a view, not tracked state.

## Layout (write this to DASHBOARD.md)

```markdown
# Control Center вАФ Dashboard

Status board across all projects, grouped by what needs attention. **Derived and
gitignored вАФ do not hand-edit.** Regenerated by `/status` and at the end of each
`/pm-loop` tick from `projects/*/tasks/*.md`. Last refreshed: <ISO 8601>.

## рЯФі Awaiting you (<n>)
_A human decision is the only thing unblocking these._
* **approve** вАФ [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј draft refined, promote `draft вЖТ ready`
* **answer** вАФ [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј Q1: <question>; Q2: <question>
* **merge** вАФ [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј <pr-url>
* **unblock** вАФ [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј <blocker reason>

## рЯЯ° In flight (<n>)
* [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј in-progress ¬Ј @<assignee> ¬Ј `<org>/<repo>`
* [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј in-review ¬Ј <pr-url>

## рЯЯҐ Next (<n>)
_Dispatchable on the next tick, or awaiting the PM's refine._
* [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј ready ¬Ј @<assignee>
* [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј draft (unrefined)

## вЫФ Blocked (<n>)
* [<task title>](/projects/<slug>/tasks/<id>.md) ¬Ј <why> ¬Ј depends_on <unmet paths>

---
Projects: <active count> ¬Ј Open tasks: <count> ¬Ј Done: <count> ¬Ј Cancelled: <count>
```

Keep a section even when empty вАФ render it with `_None._` so the board's shape is
stable (and the SessionStart hook reliably finds "Awaiting you").

## Notes
- **Report-only.** Never promote `draft вЖТ ready`, never merge, never dispatch agents
  вАФ those are `/pm-loop`'s job and the human's gates.
- **One instance.** A cross-instance ("all my groups") view is out of scope вАФ run
  `/status` inside each instance. `DASHBOARD.md` is per-instance.
- No customer PII in the board.
